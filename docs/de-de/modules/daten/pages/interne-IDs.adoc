== UnitCombination-IDs

Im Hintergrund wird beim Erstellen von Varianten jeder Kombination aus einer Einheit und einer Menge eine unitCombination-ID zugewiesen. Dabei hat die Kombination aus *1* und *Stück* immer die unitCombination-ID *1*. Alle weiteren IDs werden jedoch in der Reihenfolge vergeben, in der du Kombinationen aus Einheiten und Mengen im System speicherst. Deshalb sind unitCombination-IDs systemweit eindeutig, unterscheiden sich aber von System zu System. Welche ID welcher Kombination zugeordnet wurde, ist im plentymarkets Backend nicht ersichtlich. Diese Daten sind über die folgenden REST-Routen abrufbar:

* Alle im System vorhandenen Einheiten-Kombinationen: `GET -  items/units/combinations`
* Einheiten-Kombination mit einer bestimmten ID:  `GET - items/units/combinations/{id}`
