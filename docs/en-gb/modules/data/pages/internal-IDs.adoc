== UnitCombination IDs

When creating variations in plentymarkets, a unitCombination ID is assigned to each combination of a unit and a quantity. The combination of *1* and *Piece* always has the unitCombination ID *1*. All further IDs are assigned in the order in which you save the combinations of units and quantities in your plentymarkets system. Thus, unitCombination IDs are consistent throughout your system, but always differ from other plentymarkets systems. Which ID is assigned to which combination is not visible in the plentymarkets back end. You can access this data via the following REST routes:

* All unit combinations available in your plentymarkets system: `GET - items/units/combinations`
* Unit combinations with a particular ID: `GET - items/units/combinations/{id}`
