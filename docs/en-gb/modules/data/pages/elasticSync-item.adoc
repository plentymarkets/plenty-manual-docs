[IMPORTANT]
.Requirements
====
In order to import item data, the following data records must be present in plentysystems:

* Categories (*Item » Category* menu)
* Sales prices (*Setup » Item » Sales prices* menu)

In order to import variation data with attributes, the following data records must be present in plentysystems:

* Attributes and attribute values (*Setup » Item » Attributes* menu)
====

[TIP]
.Practical example: Importing new items
====
Imagine you’re new to plentysystems and you want to import all of your items and variations at once. How would that work? Take a look at xref:data:best-practices-elasticsync-creating-variations.adoc#[this practical example], which guides you through the process step-by-step.

=== Item ebay title

[#350]
==== Title (comma-separated)

Enter all nine eBay listing titles into the CSV file. Use commas to separate them.
This links the listing titles to the item.

*_Background info_*: Listings are retrieved on an hourly basis.
While being retrieved, the listing is assigned to an item.
xref:markets:ebay-setup.adoc#10400[Listings and items are matched] smoothly if the listing was created with plentysystems or if the item has the same name as the listing title.

If the item name differs, you can enter the listing title.
This improves the matching accuracy.
This link also makes it possible to import listings that were set up with other tools into plentysystems and have them automatically assigned to the correct item.

*_Default value_*: No default value

*_Permitted import values_*: Alpha-numeric

*_Note_*: It is not possible to only import some of the nine titles.
This is because all of the nine titles are deleted during the import process and then added again.

*_Note_*: The import is limited to a maximum of nine titles.
If your CSV file includes more than nine titles, then all of the titles over this limit will be ignored.

You can find the result of the import in the back end menu: xref:item:managing-items.adoc#45[Item » Edit item » [Open item\] » Tab: Multi-Channel » Area: eBay title]

[#590]
==== Price calculation UUID

icon:info-circle[role="blue"] xref:item:prices.adoc#1000[New price calculation]

Enter a price calculation’s UUID into the CSV file.
This applies the price calculation to whichever prices were saved for the variation.

*_Tip_*: You can copy the price calculation UUID. Open the menu *Item » Price calculation » [Open price calculation]*.
Click on the UUID in the lower left corner.
The UUID is copied to the clipboard.

*_Default value_*: No default value

*_Permitted import values_*: Alpha-numeric

*_Import the price calculation UUID last_*:
Imagine your prices are calculated with the following formula: `(PurchasePrice + TransportCosts) * 1.07`.
In this case, import the purchase price and transportation costs first and then the price calculation second.
Otherwise plentysystems won’t have all of the information it needs to correctly calculate the price.

. xref:data:ElasticSync.adoc#1500[Create a mapping], which imports the purchase price and transportation costs.
. xref:data:ElasticSync.adoc#1500[Create another mapping] in the same import. Use this one to import the price calculation UUID. +
→ The mappings are processed one after the other.

You can find the result of the import in the back end menu: xref:item:managing-items.adoc#240[Item » Edit item » [Open variation\] » Tab: Settings » Area: Sales prices]

[IMPORTANT]
.One price calculation per variation
====
Only one price calculation can be linked to a variation.
Imagine you’ve already linked a price calculation to a variation.
Then you link a second price calculation to the same variation.
In this case, the second price calculation would overwrite the first one.
Both price calculations can not simultaneously be linked to the variation.

Nonetheless, it’s still possible to calculate prices differently, e.g. in the plentyShop and on Amazon.
This is done by adding xref:item:prices.adoc#1500[multiple calculation building blocks] to your price calculation.
